全局配置 #

指Vue.use时的配置项

配置项 #

framework #

定 UI 框架，必填。类型：string

内部所有组件比如Button、Input、Select等都将使用该框架里的组件。内部已支持quasar、``element-plus`。

components #

优先级最高的组件指定方式，可指定的组件列表见下方组件表

app.use(Adminlab, {
  framework: "quasar",
  components: {
    // 指定该组件使用其它UI框架
    Button: "vuetify",
    // 指定某组件使用自定义组件
    Select: h("div"),
  },
});

componentProps #

组件通用属性配置，global可以设置全部的组件，其余部分可设置的组件列表参考下方组件表

  • global

设置组件表中所有组件的默认属性，优先级最低

  • table

设置表格模块中所用到的组件的默认属性

  • formDialog

设置弹层模块中所用到的组件的默认属性

  • searcher

设置搜索模块中所用到的组件的默认属性

  • toolbar

设置工具栏模块中所用到的组件的默认属性

app.use(Adminlab, {
  framework: "quasar",
  componentProps: {
    global: {
      Select: {
        // 将针对所有的 Select 组件
        clearable: true,
      },
    },
    table: {},
    formDialog: {},
    searcher: {},
    toolbar: {},
  },
});

组件表 #

下方注释表示组件都用在了哪些地方

下方的组件名就对应着 UI 框架中的组件，比如Form对应着quasar里的q-form

有些组件名在有些 UI 框架并没有对应的组件，比如Tag在vuetify与quasar里就不存在，这时候大家可以去对应的适配器里找（入口左侧菜单），里面会说明是用什么组件实现的

{
